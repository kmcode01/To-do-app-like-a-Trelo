#Taskboard: a Trello style task management app built with JS and Supabase. Users can register/login, then create projects, the view taskboard for each project, and create / edit / delete tasks. Tasks can be dragged and dropped between columns and reordered within a column. 

##Architecture And Tech Stack

Clasical client - server app:
   - Frontend: JS app , Bootstrap, HTML, CSS
   - Build Tools: Vite and npm
   - Backend: Supabase
   - Database: PostgresSQL
   - Authentication: Supabase Auth
   - API: Supabase REST API
   - Hosting: Nettlify
   - source code: GitHub

Use modular code structure, with separate files for different components, pages and features. Use ES6 modules to organize the code and promote reusability. Implement a clear separation of concerns, with distinct layers for data access, business logic, and presentation. Use modern JavaScript features and best practices to ensure clean, maintainable, and efficient code.

   ## UI Guidelines
   -Use HTML, css, Bootstrap and vanila JS to build the frontend.
   - Use Botstrap components and utilities to create a responsive and visually appealing design.
   - implement modern, resposive UI design, with a semantic HTML
   - Use consistent styling color scheme and typografy throught the app
   - Use appropriete icons, effects and visual cues to enhance usability and user experience.
   - Ensure the app is accessible and works well on different devices and screen sizes.
   - Use a clean and organized layout to display projects, taskboards, and tasks, making it easy for users to navigate and manage their tasks effectively.

   ## Pages and navigations
   - Split the app intor multiple pages: login registration, projects lists, taskboard, etc.
   - Implement pages as reusable components (HTML, CSS, JS)
   - Use routting to navigate between pages
   - Use full URLs like /login, /register, /projects, /project/:id/taskboard, etc. to make the app more user-friendly and SEO-friendly.

   ##Backend and database
   - Use Supabase as the backend and database for the app.  
   - Use postgreSQL as the database to store user data, projects, taskboards, and tasks.
   - Use Supabase Auth for user authentication and authorization.
   - Use RLS POLICIES to restrict access their own data based on user roles and permision.
  - Use supabase storage for file uploads (e.g task attachements)
  - When changing the database schema, always use migrations to keep track changes
- After applying mightaions in Supabase, keep a coppy of the migrations SQL file in the code.

   
   ## Authetentication and authorization
- Use Supabase Auth for user authentication and authorization.
- Implement RLS policies to restrict access to data based on user roles and permissions.
- Implement user roles with a separate table in the database table: user_roles + enum roles (admin, user, etc.)
